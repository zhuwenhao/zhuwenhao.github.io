Node.js 文件系统操作完全指南

2026-05-15 前端开发 #Node.js #JavaScript

前言

Node.js 的 fs 模块是进行文件系统操作的核心模块。无论是读取配置文件、写入日志、创建目录，还是监听文件变化，fs 模块都能胜任。本文将全面介绍 fs 模块的常用功能。

同步 vs 异步

fs 模块的所有方法都有同步和异步两个版本：

const fs = require('fs');

// 异步读取（推荐，不阻塞事件循环）
fs.readFile('file.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// 同步读取（简单场景可用）
try {
  const data = fs.readFileSync('file.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

// Promise 版本（现代推荐）
const fsPromises = require('fs/promises');
const data = await fsPromises.readFile('file.txt', 'utf8');

常用操作

文件读写

// 写入文件
await fsPromises.writeFile('output.txt', 'Hello World');

// 追加内容
await fsPromises.appendFile('output.txt', '\nNew line');

// 读取文件
const content = await fsPromises.readFile('output.txt', 'utf8');

目录操作

// 创建目录（递归）
await fsPromises.mkdir('a/b/c', { recursive: true });

// 读取目录内容
const files = await fsPromises.readdir('./');

// 检查文件/目录是否存在
const stat = await fsPromises.stat('file.txt');
console.log(stat.isFile());     // true
console.log(stat.isDirectory()); // false

文件流处理

对于大文件，使用流（Stream）可以避免一次性加载到内存：

const { createReadStream, createWriteStream } = require('fs');

// 复制大文件
const readStream = createReadStream('large-file.zip');
const writeStream = createWriteStream('copy.zip');
readStream.pipe(writeStream);

总结

在实际项目中，推荐使用 fs/promises 的 async/await 写法，代码更清晰、更不容易出错。对于大文件操作，务必使用 Stream 来避免内存溢出。
