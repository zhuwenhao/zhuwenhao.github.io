GitHub 密码认证已被移除的解决方法

2026-05-20 开发工具 #Git #GitHub

问题描述

在使用 Git 向 GitHub 推送代码时，你可能会遇到以下错误信息：

remote: Support for password authentication was removed on August 13, 2021.
fatal: Authentication failed for 'https://github.com/your-repo.git/'

这是因为 GitHub 已经移除了通过账号密码进行 Git 操作的认证方式，转而要求使用 Personal Access Token（个人访问令牌）。

解决步骤

第一步：生成 Personal Access Token

  1. 登录 GitHub，点击右上角头像 → Settings
  2. 左侧菜单最下方选择 Developer settings
  3. 选择 Personal access tokens → Tokens (classic)
  4. 点击 Generate new token (classic)
  5. 设置过期时间和权限范围（建议勾选 repo 全部权限）
  6. 点击 Generate token，立即复制保存（关闭页面后无法再看到）

第二步：使用 Token 进行认证

方法一：在推送时直接使用 Token 作为密码：

git push
# Username: 你的 GitHub 用户名
# Password: 粘贴刚才生成的 Token

方法二：将 Token 保存到 Git 凭据中（推荐）：

git config --global credential.helper store
# 下次推送时输入一次 Token 后会自动保存

第三步：配置 SSH（长期推荐方案）

如果你不想每次都输入 Token，可以配置 SSH 密钥：

# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your@email.com"

# 将公钥添加到 GitHub
# 复制 ~/.ssh/id_ed25519.pub 的内容
# GitHub → Settings → SSH and GPG keys → New SSH key

# 修改仓库远程地址为 SSH 格式
git remote set-url origin git@github.com:username/repo.git

总结

GitHub 移除密码认证是为了提高安全性。Personal Access Token 相比密码更加灵活，可以随时撤销和设置权限范围，强烈建议使用。

对于频繁使用 Git 的开发者，配置 SSH 密钥是最佳方案，一次配置，永久使用，无需反复输入密码。
